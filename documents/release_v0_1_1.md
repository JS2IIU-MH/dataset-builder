# Release v0.1.1

> リリース: v0.1.1
>
> コミット: 6b9190574c43d6b736fce6a928852ae9bfb5cfa4
> 日付: 2025-12-28

## 概要

v0.1.1 は v0.1.0 のマイナーアップデート（パッチリリース）です。本バージョンは安定性と互換性の改善、小さな不具合修正、およびいくつかのユーザビリティ向上が中心です。

## ハイライト

- データ入出力の堅牢性向上（エンコーディング/パーケット処理の改善）
- メモリ使用量と読み込みパフォーマンスの最適化（大きなCSVのチャンク読み込み対応）
- Streamlit UI の安定化（`st.cache_data` を用いたキャッシュ適用等）
- エクスポート機能の不具合修正（Parquet/CSV でのメタデータ欠落対応）
- 可視化チャートの微調整（Plotly 表示の描画崩れ修正）
- ドキュメント・リリースノートの更新

## 変更点（主なファイル/モジュール）

- `main.py`: アプリ起動処理の小さな修正（引数処理の堅牢化、メッセージ改善）
- `src/logic/data_io.py`: 読み込み時のエンコーディング判定とチャンク読み込みを追加
- `src/logic/cleaning.py`: 欠損値処理のエッジケース修正
- `src/logic/eda.py`: 相関解析の処理安定化と出力整形（NaN取り扱い修正）
- `src/logic/feature_engineering.py`: 小さなバリデーション追加
- `src/ui/charts.py`: Plotly 表示の互換性修正とレスポンシブ性改善
- `src/ui/export.py`: Parquet/CSV エクスポートの不具合修正
- `src/ui/forms.py` / `src/ui/sidebar.py`: 入力バリデーションとUIメッセージ改善

(注: 上記は本コミットでの主要変更点を概観した要約です。詳細は該当コミットの差分をご確認ください。)

## バグ修正

- 大きなCSV読み込みでのメモリ暴走を緩和するためのチャンク処理を追加
- Parquet エクスポート時にメタデータが失われるケースを修正
- 一部の非UTF8エンコーディングファイルでの読み込み失敗を改善
- UI上で稀に発生するプロット描画エラーを修正

## 既知の問題

- 非常に大きなデータセットでは依然としてメモリ負荷が高くなる可能性があります（今後の改善予定）。
- 一部の特殊なエクセルフォーマットや壊れたファイルで読み込みエラーが発生する場合があります。

## アップグレード/移行手順

特別な移行手順は不要です。既存の環境で依存関係を更新し、通常通り起動してください。

```bash
source .venv/bin/activate
pip install -r requirements.txt
streamlit run main.py
```
